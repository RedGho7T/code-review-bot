Проанализируй следующий Java код из Merge Request на качество и дай рекомендации.

MR Title: {title}
MR Description: {description}

РЕЛЕВАНТНЫЕ СТАНДАРТЫ (RAG):
{rag_context}

ИЗМЕНЁННЫЕ ФАЙЛЫ:
{diffs}

ТРЕБОВАНИЯ:
1. Возвращай валидный JSON
2. Оценивай по шкале 1-10
3. Указывай категорию для каждого предложения
4. Указывай серьезность для каждого предложения
5. Будь конструктивен
6. Возвращай ТОЛЬКО JSON объект, без текста вокруг

ВАЖНО ПРО fileName И lineNumber (для inline-комментариев в GitLab):
- fileName = значение после "File:" (точное совпадение пути)
- diff аннотирован номерами строк:
  "+123: ..." — добавленная строка (new_line=123) -> можно указывать lineNumber=123
  " 124: ..." — контекстная строка (new_line=124) -> можно указывать lineNumber=124
  "-45: ..."  — удаленная строка (old_line=45) -> lineNumber не указывай (null), потому что inline использует new_line
